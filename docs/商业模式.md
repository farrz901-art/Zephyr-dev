## 网站风格：

我会为我的产品做一个绚丽的网站（风格、动画与视觉效果参考https://www.nanfu.global/（南孚电池的国际官网），但是由于我们项目的名字是zephyr，所以要以风和气流的特效为主，再考虑到我们项目的核心是针对数据处理与连通器连接，所以我们的网页核心展示物就成了数据的拟化物（对标南孚电池的国际官网的南孚电池的在页面的各种互动展示）），这是网站的展示部分。

## 网站内容与商业逻辑：

在顶层菜单栏是（home、docs、products（产品页用于放置安装包，安装包我们在那时会打包成.exe安装包）、function、about），顶层右边是一个GO按钮和用户按钮（这就是我要注入商业逻辑的地方：设计两个安装包并在products页面中给出下载链接（base（uns-stream的基础部分和it-stream的部分）和pro（能额外使用我们的uns-stream的-extra的功能部分），pro付费下载安装（买断制），base免费下载安装），pro购买后才能下载，网页端加入用户机制，如果用户购买了pro软件，就能一直下载完全体，网页端的api后端采用便宜订阅收费或者每页多少钱（如果采用后者的付费模式，我们先给用户免费的10000页的处理（当然不包含-extra的功能），用户用完次数后，base处理采用一个合适的每页处理的价位，-extra部分的api不送免费次数，调用价格是base的三倍），如果我们在web端采用订阅付费模式，用户在首页（也就是web端的用户功能里，效仿现在国外的商业订阅模式，例如：谷歌的Gemini和openai的GPT）开启订阅后点击首页的“GO”按钮定向跳转到完整功能的网页，免费用户点击“GO”按钮只能跳转base功能的网页，其他免费用户不能直接通过链接跳转 到付费用户的网页，只能用户跳转，即"壳与核分离”的商业设计，避免强行将用户机制迁移进我们的zephyr项目中，而是将压力转移到web服务层和运维层），为了避免带宽成本和api成本高额消耗，也为了避免pro用户一人买断，多方登号下载付费版软件来逃避付费，采用机器码绑定（HWID）来尝试应对多用户逃避付费，使用反向代理鉴权防止非法调用api，使用防盗链与预签名URL解决高额带宽成本、防止安装包链接被非法公开（

为了让zephyr既能分发又不亏本，我的设计： • Base 版 (免费版)： 放在 GitHub Release（带宽免费，但用户下载慢）。 • Pro 版 (付费版)： 放在 Cloudflare R2 或 阿里云 OSS。）

安装包设计我打算和常规软件厂商一样使用在线安装器（Web Installer）：这种安装包通常只有几 MB（比如 Chrome 或 Office 的安装包）。 • 原理： 这个 .exe 本质上是一个下载器 + 校验器。它不包含zephyr的核心模型和重量级依赖。 • 安装过程：

1.   用户运行小安装包。

2.   安装包联网向我的 OSS（如 R2/S3） 请求数据。
3.   边下载边解压到用户的安装路径（安装路径采用仅为我安装（电脑用户的默认安装目录下）和为所有用户安装（用户指定安装目录））下。 • 对我的价值： * 独立设计： 不需要动zephyr源码。 • 权限控制： 可以在下载前让用户输入 License Key（Pro用户我们给他key用于解锁）。Key 不对，就不给下载那-extra的数据。 • 更新方便： 以后zephyr升级了，我们只需换云端的文件，用户的小安装包永远不用换。
