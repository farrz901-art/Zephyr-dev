# P6K Roadmap

P6K is a Zephyr-dev core hardening sequence that prepares a stronger common kernel for later Pro
and Web layers.

Roadmap:

1. `P6K-M0`: baseline audit
2. `P6K-M1`: unstructured 0.22.28 enhanced partition profiles
3. `P6K-M2`: PackageManifestV1 and artifact identity
4. `P6K-M3`: real benchmark evidence for enhanced partition profiles
5. `P6K-M4`: it-stream Airbyte-like structured sync hardening
6. `P6K-M5`: package-aware destination mapping
7. `P6K-M6`: HTML Composer Node
8. `P6K-M7`: Chunker Node and ZephyrChunk
9. `P6K-M8`: Embedder Node and EmbeddingProvider
10. `P6K-M9`: batch workflow runtime minimum
11. `P6K-M10`: Pro/Web SDK and API surface
12. `P6K-M11`: Chinese benchmark suite
13. `P6K-M12`: LLMProvider / Extract / NER / Table Description
14. `P6K-M13`: DeepSeek-OCR2 fallback experiment
15. `P6K-M14`: Feishu UNS and IT sources

Roadmap safety rules:

- Zephyr-dev remains the pure core
- Zephyr-base is not modified by P6K milestone work
- commercial logic stays outside Zephyr-dev
- Web-core remains the place for future entitlement, license, billing, and gated download logic
- P6K is not a cloud or SaaS runtime line
