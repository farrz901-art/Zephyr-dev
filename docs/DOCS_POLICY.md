# Docs Policy

- `AGENTS.md` files are the authoritative rules and architecture surfaces.
- Current-phase docs in `docs/` provide planning, handoff, and context; they are not
  implementation SSOT over code, contracts, and tests.
- `docs/archive/legacy-p0-p4/` is historical reference only.
- `docs/review-needed/` contains older docs awaiting review and is not current truth by default.
- Every phase-state doc should name its current stage explicitly.
- Avoid duplicate topic docs with different naming or casing.
- When a doc becomes historical, archive it rather than letting it compete with current truth.
