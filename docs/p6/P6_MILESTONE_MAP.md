# P6 Milestone Map

## P6-M0: P6 Plan Freeze and Execution Control

- freeze P6 phase boundaries
- freeze repo roles and review policy
- freeze export-manifest expectations
- mark hard gates before productization expands

## P6-M1: Six-Repo Boundary and Pure-Core Contamination Guard

- define Zephyr-dev pure-core boundary
- define derivation policy
- define denylist and contamination scanner
- define first review-gate expectations

## P6-M2: Public/Private Core Export and Derivation Mechanics

- create export-manifest rules
- define allowed public exports from Zephyr-dev
- define protected private overlays
- define release lineage from Zephyr-dev SHA to downstream repos

## P6-M3: Base Free Desktop Product MVP+

MVP+ here means a bounded but product-real starting point, not a throwaway
demo.

- public desktop shell
- local file selection flow
- run-state animation and progress UX
- success/failure receipt display
- error-diagnosis panel

Design window input required: yes.

## P6-M4: Pro Private Desktop Product MVP+

- private desktop shell
- entitlement-client status surface
- device-state presentation
- Pro-only feature entry points
- offline grace and device-limit visual states

Design window input required: yes.

## P6-M5: Web-core Commercial Control Plane MVP+

- bounded commercial control plane
- entitlement and download/update authority surfaces
- review-gated private APIs
- export-manifest and release-lineage tracking

Design window input required: no.

## P6-M6: Web Frontend Entry and Bounded Web/API Demo

- login and register entry flows
- dashboard
- download and purchase entry points
- bounded web/API task submission
- quota and usage display
- result view

Design window input required: yes.

## P6-M7: Site, Legal, Brand, and Marketing Foundation

- homepage
- pricing page
- product comparison page
- go-button section
- enterprise cooperation form
- brand visual system
- logo usage
- product screenshots
- landing animation direction

Design window input required: yes.

## P6-M8: Signing, Download Control, and Update Authorization Gates

- update-available UX
- download-authorized UX
- download-rejected UX
- signature-verified UX
- update-entitlement-expired UX
- release and package distribution hard gates

Design window input required: yes.

## P6-M9: P6 End-to-End Commercial Validation

- bounded end-to-end product validation
- public/private release-chain verification
- commercial-boundary regression checks
- final blocker burn-down

## P6-M10: P6 Final Release Candidate and Archive

- final release candidate handoff
- final release pack
- archive of boundary, export, and review evidence
- downstream planning inputs for P7

## P7 Deferred Themes

Unless explicitly re-authorized earlier, treat these as P7 themes:

- Kubernetes and Helm packaging claims
- broader cloud deployment claims
- multi-tenant SaaS platform ownership
- enterprise RBAC and enterprise permission systems
- advanced anti-crack and anti-tamper systems
- complex financial reconciliation
